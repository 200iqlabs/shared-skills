# The Five - Checklisty operacyjne

## 1. Founders Agreement Checklist

### Przed podpisaniem umowy wspólniczej przejdź przez wszystkie punkty:

#### A. Kto wnosi co?
- [ ] Czas: ile godzin tygodniowo każdy wspólnik? (pełny etat / po godzinach / doradczo)
- [ ] Pieniądze: kto wnosi kapitał? W jakiej formie? (pożyczka / wkład własny / konwersja długu)
- [ ] IP: czy ktoś wnosi istniejący kod/rozwiązania? Jak wycenione?
- [ ] Umiejętności: jakie unikalne kompetencje wnosi każdy? Jak się uzupełniacie?
- [ ] Sieć kontaktów: kto ma dostęp do klientów/inwestorów/ekspertów?

#### B. Jak dzielicie udziały?
- [ ] Podział odzwierciedla realne wkłady (czas + pieniądze + IP)?
- [ ] Czy jest mechanizm vestingu/odwróconego vestingu?
- [ ] Kto ma pakiet kontrolny (>50%)?
- [ ] Czy są udziały zarezerwowane na przyszłych pracowników (ESOP)?

#### C. Co jeśli ktoś odejdzie?
- [ ] Jaki jest okres wypowiedzenia?
- [ ] Co dzieje się z udziałami przy dobrowolnym odejściu?
- [ ] Co przy wyrzuceniu (bad leaver vs good leaver)?
- [ ] Po jakiej cenie odkup udziałów?
- [ ] Co w razie śmierci/długotrwałej choroby?
- [ ] Czy jest zakaz konkurencji?

#### D. Co znaczy sukces, a co porażka?
- [ ] Jasne cele na 1 rok (przychody/użytkownicy/PMF)?
- [ ] Jasne cele na 3 lata?
- [ ] Kiedy uznajecie, że projekt się nie udał?
- [ ] Co wtedy? (pivot / zamknięcie / sprzedaż)

#### E. Role i odpowiedzialności
- [ ] Kto odpowiada za: produkt / sprzedaż / finanse / HR / marketing?
- [ ] Kto podejmuje decyzje operacyjne?
- [ ] Kto ma ostatnie słowo przy sporach?
- [ ] Od jakich kwot wymagana jest zgoda obu stron?
- [ ] Jak często spotkania wspólników?

## 2. Due Diligence Checklist (kupujący udziały)

### Przed kupnem udziałów sprawdź:

#### A. Produkt i technologia
- [ ] Demo produktu - czy działa jak obiecują?
- [ ] Architektura techniczna - czy skalowalna?
- [ ] Tech debt - ile trzeba będzie przepisać?
- [ ] Bezpieczeństwo - czy były audyty?
- [ ] Własność IP - kto jest właścicielem kodu?

#### B. Rynek i klienci
- [ ] Ile użytkowników? (aktywnych vs zarejestrowanych)
- [ ] Ile płacących klientów?
- [ ] Jaki churn? (miesięczny/roczny)
- [ ] NPS lub inne miary satysfakcji?
- [ ] Referencje - porozmawiaj z 3+ klientami!
- [ ] Wielkość rynku (TAM/SAM/SOM)?

#### C. Finanse
- [ ] Przychody ostatnie 12 miesięcy (MRR/ARR)
- [ ] Burn rate miesięczny
- [ ] Runway - na ile miesięcy starczy kasy?
- [ ] Struktura kosztów (ludzie / infra / marketing)
- [ ] Zobowiązania i długi
- [ ] Historia finansowania (rundy, wyceny)

#### D. Zespół
- [ ] Kim są kluczowe osoby?
- [ ] Jak długo pracują w firmie?
- [ ] Czy mają umowy o pracę / B2B?
- [ ] Czy są klauzule zakazu konkurencji?
- [ ] Kto odszedł w ostatnim roku i dlaczego?

#### E. Prawne
- [ ] Forma prawna spółki
- [ ] Struktura udziałowa - kto ma ile?
- [ ] Czy są ciągnące zobowiązania (drag-along, tag-along)?
- [ ] Umowy z klientami - jakie warunki?
- [ ] Czy są toczące się spory?

#### F. Red flags (każde TAK = głębsza analiza)
- [ ] Founder/wspólnicy odchodzą lub zmniejszają zaangażowanie?
- [ ] Kluczowi pracownicy odeszli w ostatnich 6 miesiącach?
- [ ] Zmiana strategii >2 razy w ostatnim roku?
- [ ] Brak dokumentacji technicznej?
- [ ] Niejasna struktura własności IP?
- [ ] Konflikty między wspólnikami?

## 3. Product-Market Fit Checklist

### Sygnały, że masz PMF:

#### Twarde metryki
- [ ] Retention >80% po 3 miesiącach
- [ ] NPS >50
- [ ] Organiczne polecenia (>30% nowych klientów)
- [ ] Klienci płacą bez długich negocjacji
- [ ] Upgrade rate >20% (klienci kupują więcej)
- [ ] Churn <5% miesięcznie

#### Miękkie sygnały
- [ ] Klienci dzwonią z prośbą o więcej funkcji (nie o niższą cenę)
- [ ] Klienci sami opisują twój produkt znajomym
- [ ] Trudno nadążyć z supportem (dobry problem!)
- [ ] Sprzedaż staje się powtarzalna, nie przypadkowa
- [ ] Klienci protestują, gdy coś zmienisz

#### Test Sean'a Ellisa
Pytanie: "Jak bardzo byłbyś rozczarowany, gdyby ten produkt przestał istnieć?"
- >40% "bardzo rozczarowany" = masz PMF
- <40% = szukaj dalej

#### Red flags - NIE masz PMF jeśli:
- [ ] Każda sprzedaż wymaga customizacji
- [ ] Klienci chwalą, ale nie płacą
- [ ] Retention <60% po 3 miesiącach
- [ ] Większość leadów przychodzi od Ciebie, nie organicznie
- [ ] Musisz tłumaczyć dlaczego produkt jest potrzebny

## 4. Scaling Readiness Checklist

### Przed skalowaniem sprawdź:

#### A. Silnik wzrostu
- [ ] Masz powtarzalny sposób zdobywania klientów?
- [ ] Ten kanał jest skalowalny (nie tylko Ty)?
- [ ] Znasz CAC (Customer Acquisition Cost)?
- [ ] CAC < LTV/3?
- [ ] Czas zwrotu CAC < 12 miesięcy?

#### B. Oferta
- [ ] Produkt jest standaryzowany (nie każdy projekt na miarę)?
- [ ] Onboarding jest samoobsługowy lub skalowalny?
- [ ] Możesz obsłużyć 10x więcej klientów bez 10x więcej ludzi?
- [ ] Masz dokumentację/FAQ/helpdesk?

#### C. Zespół
- [ ] Kluczowi ludzie wiedzą, że planujesz skalowanie?
- [ ] Mają apetyt na wzrost (szansa vs zagrożenie)?
- [ ] Masz plan rekrutacji na najbliższe 6 miesięcy?
- [ ] Kultura jest spisana i zrozumiała?

#### D. Ty jako founder
- [ ] Możesz delegować decyzje operacyjne?
- [ ] Masz zaufanych ludzi na kluczowych stanowiskach?
- [ ] Nie jesteś wąskim gardłem w żadnym procesie?
- [ ] Masz czas na strategię, nie tylko gaszenie pożarów?

#### E. Finanse
- [ ] Masz 12-18 miesięcy runway?
- [ ] Wiesz ile kosztuje pozyskanie 1 klienta?
- [ ] Masz model finansowy na najbliższe 12 miesięcy?
- [ ] CFO/księgowość daje prognozy na czas?

## 5. Weekly Review Checklist

### Co tydzień przejdź przez:

#### A. Metryki (5 min)
- [ ] MRR vs cel
- [ ] Nowi klienci vs cel
- [ ] Churn w tym tygodniu
- [ ] Pipeline sprzedażowy

#### B. Priorytety (10 min)
- [ ] Co było #1 priorytetem w tym tygodniu?
- [ ] Czy został zrealizowany?
- [ ] Jeśli nie - dlaczego?
- [ ] Co jest #1 priorytetem na przyszły tydzień?

#### C. Zespół (5 min)
- [ ] Kto potrzebuje wsparcia?
- [ ] Czy są konflikty do rozwiązania?
- [ ] Czy ktoś jest na granicy wypalenia?

#### D. Finanse (5 min)
- [ ] Cash na koncie
- [ ] Runway
- [ ] Czy coś niepokojącego w tym tygodniu?

#### E. Ja jako founder (5 min)
- [ ] Jak się czuję? (1-10)
- [ ] Czy miałem czas na strategię, nie tylko operacje?
- [ ] Czy coś mnie blokuje?
- [ ] Co mogę zdelegować?

## 6. Tornado Identification Framework

### Jak rozpoznać tornado (rosnący trend)?

#### Sygnały zewnętrzne
- [ ] VC wkładają pieniądze w tę przestrzeń (sprawdź Crunchbase)
- [ ] Big tech ogłasza produkty w tym obszarze
- [ ] Media branżowe piszą regularnie
- [ ] Konferencje dedykowane temu tematowi
- [ ] Job postings rosną (sprawdź LinkedIn)

#### Sygnały z rynku
- [ ] Klienci sami pytają o to rozwiązanie
- [ ] RFP zawierają te wymagania
- [ ] Konkurencja się pojawia (dobry znak!)
- [ ] Consultanci tworzą praktyki w tym obszarze

#### Twoja przewaga
- [ ] Masz wiedzę/doświadczenie w tym obszarze?
- [ ] Masz dostęp do klientów, którzy tego potrzebują?
- [ ] Możesz być w top 5% ekspertów?
- [ ] Timing jest dobry (nie za wcześnie, nie za późno)?

#### Red flags - to NIE jest tornado
- [ ] Tylko startupy się tym interesują (nie enterprise)
- [ ] Technologia jest 5+ lat na rynku bez przełomu
- [ ] Regulacje blokują adopcję
- [ ] Nie ma wyraźnego przypadku użycia (rozwiązanie szuka problemu)

---

## Jak używać tych checklist

1. **Przed ważną decyzją** - przejdź przez odpowiednią checklistę
2. **Przy rozmowie z asystentem** - powołuj się na konkretne punkty
3. **Co tydzień** - Weekly Review Checklist
4. **Co kwartał** - pełny przegląd wszystkich obszarów

Pamiętaj: "Większość firm popełnia te same błędy. Nie dlatego, że brakuje im talentu, ale dlatego, że nikt nie daje im sprawdzonego przepisu." - Tomasz Karwatka
