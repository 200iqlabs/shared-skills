# Kontekst podmiotów

## PLSoft — Jednoosobowa Działalność Gospodarcza (JDG)

| Parametr | Wartość |
|----------|---------|
| Forma prawna | Jednoosobowa działalność gospodarcza |
| Właściciel | Paweł Lipowczan |
| Profil | Usługi IT, automatyzacje, konsulting, integracja AI |
| Model biznesowy | B2B, klienci od małych firm po korporacje |
| Odpowiedzialność | Pełna, osobista, całym majątkiem |

### Potrzeby prawne PLSoft
- Ogólne Warunki Współpracy (OWU) — do stworzenia
- Wzory umów z klientami (B2B, NDA, przeniesienie IP)
- Polityka prywatności i dokumentacja RODO
- Ochrona własności intelektualnej (kod, automatyzacje, know-how)
- Umowy z podwykonawcami/kontraktorami

## 200IQ Labs Prosta Spółka Akcyjna (PSA)

| Parametr | Wartość |
|----------|---------|
| Forma prawna | Prosta Spółka Akcyjna (KSH art. 300¹+) |
| Rola użytkownika | Akcjonariusz + Członek Zarządu |
| Status | Operacyjna (~6 miesięcy), walidowany produkt, pierwsi klienci |
| Faza | Skalowanie, akwizycja klientów, refinement produktu |
| Odpowiedzialność | Co do zasady ograniczona do wkładu (ale odpowiedzialność zarządu za zobowiązania w określonych warunkach) |

### Potrzeby prawne 200IQ Labs
- Founders Agreement — w przygotowaniu
- Relacje akcjonariuszy (nabywanie dodatkowych akcji planowane)
- Umowa spółki / statut — kluczowy dokument do analizy
- IP — ochrona produktu, prawa autorskie do kodu
- Regulamin zarządu
- ToS i polityka prywatności produktu
- Umowy z kontrahentami i współpracownikami

### Specyfika PSA
PSA to stosunkowo nowa forma prawna (od 2021). Kluczowe cechy wpływające na analizy:
- Akcje nie mają wartości nominalnej
- Wkładem może być praca/usługi
- Uproszczone podwyższenie kapitału
- Elektroniczny rejestr akcjonariuszy
- Ograniczone orzecznictwo — przy interpretacjach sygnalizuj tę niepewność

## Relacje między podmiotami

### PLSoft → 200IQ Labs (usługi B2B)
PLSoft może świadczyć usługi IT/konsultingowe dla 200IQ Labs. Wymaga to:
- Formalnej umowy B2B (unikanie konfliktu interesów — Paweł jest po obu stronach)
- Warunków rynkowych (arm's length) — szczególnie ważne przy transakcjach z podmiotem powiązanym
- Zgodności z KSH dot. czynności z zainteresowanym członkiem zarządu

### Transfer IP
- Kod/oprogramowanie tworzone przez Pawła — czyje jest? Zależy od umowy
- Jeśli Paweł tworzy dla 200IQ Labs jako B2B z PLSoft → potrzebna klauzula przeniesienia IP
- Jeśli tworzy jako członek zarządu → domyślnie może nie przechodzić na spółkę (brak stosunku pracy)
- Wymaga formalnego uregulowania w umowie

### RODO — role
- PLSoft jako administrator danych swoich klientów
- 200IQ Labs jako administrator danych użytkowników produktu
- Gdy PLSoft przetwarza dane dla 200IQ Labs → umowa powierzenia (PLSoft jako procesor)
- Oddzielne polityki prywatności dla każdego podmiotu

### Potencjalne konflikty interesów
- Paweł jest po obu stronach relacji PLSoft ↔ 200IQ Labs
- KSH reguluje transakcje z zainteresowanym członkiem zarządu
- Konieczne: transparentność, warunki rynkowe, dokumentacja decyzji
- W zależności od statutu — możliwy wymóg zgody wspólników/rady na transakcje powiązane
