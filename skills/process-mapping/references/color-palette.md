# Color Palette for Process Maps

## Color Loading Priority

Before generating any diagram, check for colors in this order:

1. **Project design system** — look for `design-system.md` or `design-tokens.css` in the project root (generated by vibe-coding skill). Map project colors to process semantics.
2. **Context overrides** — check `context/process-mapping.md` for explicit color mappings.
3. **Default palette** — use the colors defined below.

If a project design system exists, map its tokens to process semantics:
- Primary color → Process step stroke/fill
- Accent color → Start/Trigger
- Success color → End/Completion
- Warning color → Decision/Gateway
- Error color → Risk/Warning
- Secondary/Muted color → Arrows, labels

## Default Semantic Palette

### Process Elements

| Element | Stroke | Background | When to Use |
|---------|--------|------------|-------------|
| **Start/Trigger** | `#c2410c` | `#fed7aa` | First step, process entry point |
| **End/Success** | `#047857` | `#a7f3d0` | Final step, successful outcome |
| **Process Step** | `#1e3a5f` | `#dbe4ff` | Regular manual process step |
| **Automated Step** | `#6d28d9` | `#ddd6fe` | Automated/system step (TO-BE) |
| **Decision/Gateway** | `#b45309` | `#fef3c7` | Branch point, condition check |
| **Risk/Warning** | `#b91c1c` | `#fecaca` | Problem area, bottleneck |
| **External System** | `#495057` | `#e9ecef` | External service, third-party |

### Connectors

| Element | Color | Width | Style |
|---------|-------|-------|-------|
| **Main flow arrow** | `#495057` | 2 | solid |
| **Decision branch** | `#868e96` | 1.5 | solid |
| **Optional/alternative** | `#adb5bd` | 1 | dashed |
| **Swim lane divider** | `#ced4da` | 1 | dashed |

### Text

| Purpose | Color | Size |
|---------|-------|------|
| **Section title** (AS-IS, TO-BE) | `#1e3a5f` | 20px |
| **Block action name** | inherit from container stroke | 14px |
| **Block actor/tool** | inherit from container stroke | 14px |
| **Arrow label** (Yes/No) | `#495057` | 12px |
| **Swim lane label** | `#868e96` | 12px |
| **Annotation** | `#868e96` | 11px |

### AS-IS vs TO-BE Differentiation

When showing both AS-IS and TO-BE:

| Map | Background tint | Purpose |
|-----|----------------|---------|
| **AS-IS** | Use default palette as-is | Current state |
| **TO-BE changed steps** | Use Automated Step colors (`#6d28d9`/`#ddd6fe`) | Steps that changed |
| **TO-BE unchanged steps** | Use Process Step colors with lower opacity stroke | Steps that stay the same |
| **TO-BE removed steps** | Use Risk/Warning colors with dashed stroke | Steps being eliminated |

## AppState

```json
{
  "viewBackgroundColor": "#ffffff",
  "gridSize": 20
}
```

For dark-themed projects, use:
```json
{
  "viewBackgroundColor": "#1e1e1e",
  "gridSize": 20
}
```

## Font Settings

All text uses monospace for consistency:
```json
{
  "fontFamily": 3,
  "lineHeight": 1.25
}
```

`fontFamily: 3` = Cascadia/monospace in Excalidraw.
